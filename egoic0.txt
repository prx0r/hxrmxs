## Abstract

Contemporary therapeutic AI systems suffer from fundamental limitations: static response patterns, insufficient personalization, and absence of validated self-improvement mechanisms. The HelloKoa Unified Intelligence Platform addresses these challenges by introducing a novel architecture that combines deep user profiling, transparent reasoning, and dual-loop autonomous learning to create a therapeutic AI that is personalized, clinically-grounded, and capable of validated self-improvement.

The platform is built upon four core architectural pillars: (1) A Dynamic Modular Profile Vector Network (DMPVN) employing quality-aware transformers and temporal convolutional networks for multi-modal user profiling; (2) A deliberative, multi-step reasoning engine ensuring transparent and evidence-based response generation; (3) A multi-layered autonomous learning system combining intrinsic reward-driven optimization with expert validation loops; and (4) An event-driven, modular architecture designed for scalability and clinical robustness.

This whitepaper provides the complete technical blueprint for the HelloKoa system, detailing its algorithms, data flows, machine learning methodologies, and clinical integration patterns. The architecture establishes a new paradigm for responsible AI-driven mental health tools that continuously evolve while maintaining clinical efficacy and safety standards.

**Keywords:** Therapeutic AI, Dynamic User Profiling, Autonomous Learning, Clinical Validation, Graph Neural Networks, Multi-Step Reasoning

## 1. Introduction

Current therapeutic AI systems operate on static models that fail to adapt to individual user needs or evolve based on clinical outcomes. These limitations manifest in three critical areas: (1) insufficient personalization due to rigid user representations, (2) opaque decision-making processes that lack clinical transparency, and (3) absence of validated self-improvement mechanisms that ensure continuous alignment with therapeutic best practices.

The HelloKoa Unified Intelligence Platform represents a paradigm shift toward adaptive therapeutic AI through the implementation of two distinct learning loops. The first is an intrinsic, performance-driven loop that enables autonomous model refinement based on uncertainty reduction and competence gains. The second is an external, expert-guided "wisdom" loop that validates AI responses against curated expert therapeutic dialogues, ensuring clinical grounding and adherence to established therapeutic principles.

This architecture addresses the fundamental challenge of creating therapeutic AI that maintains the delicate balance between personalization and clinical efficacy while providing mechanisms for validated, transparent improvement over time.

## 3. The HelloKoa Workflow: A Four-Phase Process

### Phase 1: Profile Initialization & Vector Construction

#### Step 1.1: User Onboarding & Assessment

User registration initiates with the creation of a `User` entity in PostgreSQL, followed by the administration of validated psychometric assessments (PHQ-9, GAD-7, custom therapeutic evaluations). Assessment results are stored as `UserAssessment` records with JSON-serialized scores and confidence ratings.

#### Step 1.2: DMPVN Computation

**Description:** The `DMPVNService` computes a dynamic, 32-dimensional user personality vector through heterogeneous graph neural network processing.

**Subprocesses:**

1. **Assessment Embeddings:** The `QualityAwareTransformer` processes psychometric scores using a 2-layer transformer encoder with quality-weighted attention:

```python
class QualityAwareTransformer(nn.Module):
    def forward(self, x: torch.Tensor, quality_weights: torch.Tensor) -> torch.Tensor:
        # Quality-weighted input processing
        weighted_x = x * quality_weights.unsqueeze(-1)
        
        # Transformer encoding with attention
        encoded = self.encoder(weighted_x)
        
        # Global average pooling and projection
        pooled = encoded.mean(dim=1)
        return self.fc(self.dropout(pooled))
```

2. **Conversational Context:** The `TemporalConvolutionalNetwork` (TCN) processes the last 30 conversation messages through a 3-layer CNN with dilated convolutions and residual connections:

```python
def forward(self, x: torch.Tensor) -> torch.Tensor:
    x = x.transpose(1, 2)  # [batch, features, sequence]
    
    # Layer 1: dilation=1
    conv1_out = self.relu(self.conv1(x))
    if self.residual1:
        conv1_out = conv1_out + self.residual1(x)
    
    # Layer 2: dilation=2  
    conv2_out = self.relu(self.conv2(conv1_out))
    conv2_out = conv2_out + conv1_out  # Residual connection
    
    # Layer 3: dilation=4
    conv3_out = self.conv3(conv2_out)
    return conv3_out.mean(dim=2)  # Global average pooling
```

3. **Graph Structure:** A `HeteroConv` GNN integrates assessment and conversational embeddings through graph attention with dynamic edge modulation:

**Implementation:** Core DMPVN computation logic:

```python
async def compute_user_vector(self, user_id: str, session: AsyncSession, 
                             conversation_history: List[str] = None) -> torch.Tensor:
    # Fetch user assessments
    assessments = await self._get_user_assessments(session, user_id)
    
    # Build heterogeneous graph structure
    data = HeteroData()
    data["user"].x = torch.zeros(1, 32, device=self.device)
    
    if assessments:
        # Process assessment scores with quality weights
        scores_tensor = torch.tensor([a.scores_list for a in assessments])
        quality_weights = torch.tensor([self._get_quality_weight(a) for a in assessments])
        
        # Generate assessment embeddings
        data["assessment"].x = self.assessment_embedder(scores_tensor, quality_weights)
        
        # Create graph edges with confidence weighting
        edge_index, edge_attr = self._build_graph_edges(assessments)
        
        # Apply conversational context modulation
        if conversation_history:
            context_modulation = self._compute_context_modulation(conversation_history)
            edge_attr *= context_modulation
    
    # Heterogeneous graph convolution
    with torch.no_grad():
        out_dict = self.hetero_conv(data.x_dict, data.edge_index_dict)
        user_vector = out_dict["user"][0]
    
    return user_vector
```

User messages trigger the creation of `UserInteractionEvent` records, which are atomically stored in PostgreSQL and queued in Redis for asynchronous processing. The `EventProcessor` implements a fault-tolerant consumption pattern with dead letter queue handling for failed processing attempts.

#### Step 2.3: Safety Arbitration

**Description:** The `SafetyGovernor` implements a two-tier safety validation system before response delivery.

**Subprocesses:**

1. **NLI-based Contradiction Detection:** Uses cross-encoder/nli-distilroberta-base to detect factual contradictions against retrieved knowledge with configurable threshold (default: 0.5).

2. **Crisis Detection:** Keyword-based juncture classification with confidence scoring:

```python
def detect_juncture(self, text: str) -> Tuple[int, float]:
    crisis_keywords = ["harm", "hurt", "kill", "suicide", "die", "death", 
                      "end it", "give up", "hopeless", "worthless"]
    
    crisis_count = sum(1 for keyword in crisis_keywords if keyword in text.lower())
    confidence = min(crisis_count * 0.3, 1.0)
    
    if confidence >= 0.5:
        return 3, confidence  # Crisis
    elif confidence >= 0.3:
        return 2, confidence  # Elevated  
    else:
        return 1, confidence  # Normal
```

All safety decisions are logged to the `SafetyAudit` table with full context preservation for regulatory compliance and system monitoring.

### Phase 3: Insight Generation

The platform generates system-level insights through continuous analysis of aggregated safety and interaction data. The `SafetyGovernor.monitor_metrics()` method implements real-time monitoring of critical therapeutic indicators:

- **Crisis Rate:** Percentage of interactions triggering crisis protocols
- **Contradiction Rate:** Frequency of factual inconsistencies in responses
- **User Satisfaction:** Derived from feedback patterns and engagement metrics
- **Response Quality:** Statistical analysis of therapeutic effectiveness measures

These metrics enable proactive system adjustments and clinical oversight integration.

### Phase 4: Iteration & System Refinement (The Learning Loops)

#### Step 4.1: The Autonomous Learning Loop (Self-Improvement)

**Description:** The autonomous learning system implements condition-triggered training based on intrinsic reward calculation, moving beyond static retraining schedules to dynamic, performance-driven optimization.

**Subprocesses:**

1. **Trigger Condition:** The `EnhancedWeightedTrainer.should_start_training()` method calculates intrinsic reward from uncertainty reduction and competence gains:

```python
def compute_reward(self, uncertainty_before: float, uncertainty_after: float, 
                  competence_gain: float) -> float:
    uncertainty_reduction = max(0, uncertainty_before - uncertainty_after)
    reward = self.alpha * uncertainty_reduction + self.beta * competence_gain
    return reward
```

Where α = 0.5 (uncertainty weight) and β = 0.5 (competence weight).

2. **Training:** When intrinsic reward exceeds threshold (default: 0.5), the system employs a combined loss function:

**Mathematical Implementation:**

```
L_total = L_supervised + λ * R_intrinsic

Where:
- L_supervised: Traditional supervised learning loss (MSE)
- λ: Intrinsic reward weighting factor (default: 0.5)  
- R_intrinsic: Calculated intrinsic reward signal
```

3. **Validation:** The `StatisticalValidator` employs paired t-tests to assess model improvement significance:

```python
async def validate(self, new_adapter_name: str):
    pre_scores, post_scores = await self._collect_validation_data(new_adapter_name)
    
    if len(pre_scores) >= 3 and len(post_scores) >= 3:
        stat, p_value = ttest_rel(post_scores, pre_scores)
        
        # Deploy only if statistically significant improvement
        if p_value < self.significance_threshold and stat > 0:
            await self._deploy_adapter(new_adapter_name)
```

4. **Deployment:** The `ModelRegistry` implements hash-based A/B testing for gradual model rollout, ensuring system stability during transitions.

#### Step 4.2: The Golden Standard Refinement Loop (Learning Wisdom)

**Description:** The offline validation process, driven by `scripts/validate_with_golden_standards.py`, ensures clinical grounding through expert dialogue comparison.

**Subprocesses:**

1. **Simulation:** The script processes curated expert-client dialogues, sending client messages to the live HelloKoa API to generate AI responses for comparison.

2. **Critique:** Using the `critique_and_correct_prompt.txt`, the LLM performs clinical supervisor analysis:

3. **Corrective Data Generation:** High-quality training examples are automatically generated with expert responses as targets, quality scores of 9.5, and detailed improvement metadata.

**Impact:** This loop ensures the AI's evolution remains grounded in clinical best practices, preventing drift toward mere user satisfaction optimization at the expense of therapeutic efficacy.

## 5. Performance and Scalability Considerations

### 5.1 Computational Efficiency

**Graph Neural Network Optimization:** The DMPVN employs efficient sparse matrix operations and batched processing to minimize computational overhead while maintaining personalization quality.

**Caching Strategy:** Redis-based caching of user vectors, therapeutic components, and frequent LLM responses reduces response latency to sub-second levels.

**Asynchronous Processing:** Event-driven architecture with async/await patterns ensures high concurrency without blocking therapeutic interactions.

### 5.2 Scalability Architecture

**Horizontal Scaling:** Microservices design enables independent scaling of computation-intensive components (DMPVN, LLM processing) from data-intensive operations (event processing, safety monitoring).

**Database Optimization:** PostgreSQL with proper indexing strategies and connection pooling supports thousands of concurrent therapeutic sessions.

**Vector Database Scaling:** ChromaDB's distributed architecture enables semantic search scaling across large therapeutic component libraries.

    def embed(self, text: str) -> list:
        """Mock embedding model that returns consistent embeddings"""
        # Simple hash-based embedding for consistency
        import hashlib
        hash_obj = hashlib.md5(text.encode())
        hash_hex = hash_obj.hexdigest()
        
        # Convert hex to normalized float array
        embedding = []
        for i in range(0, min(len(hash_hex), 96), 2):  # 384 dimensions / 4 = 96 hex pairs
            hex_pair = hash_hex[i:i+2] if i+1 < len(hash_hex) else hash_hex[i] + "0"
            embedding.extend([int(hex_pair, 16) / 255.0] * 4)
        
        # Pad to 384 dimensions
        while len(embedding) < 384:
            embedding.append(0.1)
        
        return embedding[:384]

## src/utils/logging.py
```python
import logging
import sys
from typing import Optional

def get_logger(name: str, level: Optional[str] = None) -> logging.Logger:
    """Get a configured logger instance"""
    logger = logging.getLogger(name)
    
    if not logger.handlers:
        # Set level
        log_level = getattr(logging, (level or "INFO").upper(), logging.INFO)
        logger.setLevel(log_level)
        
        # Create handler
        handler = logging.StreamHandler(sys.stdout)
        handler.setLevel(log_level)
        
        # Create formatter
        formatter = logging.Formatter(
            '%(asctime)s - %(name)s - %(levelname)s - %(message)s',
            datefmt='%Y-%m-%d %H:%M:%S'
        )
        handler.setFormatter(formatter)
        
        # Add handler to logger
        logger.addHandler(handler)
    
    return logger
```

class EventStore:

    async def append_event(self, event_data: Dict[str, Any]) -> str:
        """Store event in database and queue for processing"""
        try:
            async with AsyncSession(self.db_engine) as session:
                # Ensure user exists
                user_stmt = select(User).where(User.id == event_data["user_id"])
                user_result = await session.execute(user_stmt)
                user = user_result.scalar_one_or_none()

                    session.add(user)
                    await session.flush()
                
                # Create event
                event = UserInteractionEvent(
                    user_id=event_data["user_id"],
                    message=event_data["message"]
                )
                session.add(event)
                await session.commit()
                
                # Queue for processing
                await self.redis_client.lpush("event_queue", event.id)
                logger.info(f"Appended event {event.id} for user {event_data['user_id']}")
                
                return event.id
                
        except Exception as e:
            logger.error(f"Failed to append event: {e}")
            raise

class EventProcessor:
    def __init__(self, event_store: EventStore, context):
        self.event_store = event_store
        self.context = context
        self._running = False

    async def process_events(self):
        """Main event processing loop"""
        self._running = True
        logger.info("Starting event processor")
        
        while self._running:
            try:
                # Get event from queue with timeout
                event_id_bytes = await self.event_store.redis_client.brpop("event_queue", timeout=1)
                
                if event_id_bytes:
                    _, event_id = event_id_bytes
                    event_id = event_id.decode('utf-8')
                    
                    async with AsyncSession(self.event_store.db_engine) as session:
                        # Get event
                        stmt = select(UserInteractionEvent).where(UserInteractionEvent.id == event_id)
                        result = await session.execute(stmt)
                        event = result.scalar_one_or_none()
                        
                        if event and event.processed == "pending":
                            try:
                                await self._handle_event(event)
                                event.processed = "processed"
                                await session.commit()
                                logger.info(f"Successfully processed event {event_id}")
                                
                            except Exception as e:
                                logger.error(f"Failed to process event {event_id}: {e}")
                                event.processed = "failed"
                                await session.commit()
                                # Move to dead letter queue
                                await self.event_store.redis_client.lpush("dead_letter_queue", event_id)
                
            except Exception as e:
                logger.error(f"Event processing loop error: {e}")
                await asyncio.sleep(1)

        """Handle individual event processing"""
        try:
            # Generate response using multi-step reasoning
            response = await self.context.multi_step_reasoning.generate_response(
                event.user_id, 
                event.message
            )
            
            # Store conversation history
            async with AsyncSession(self.event_store.db_engine) as session:
                from src.database.models import ConversationHistory
                conversation = ConversationHistory(
                    user_id=event.user_id,
                    message=event.message,
                    response=response
                )
                session.add(conversation)
                await session.commit()
            
            logger.info(f"Generated response for user {event.user_id}: {response[:100]}...")
            
        except Exception as e:
            logger.error(f"Failed to handle event {event.id}: {e}")
            raise

        """Stop the event processor"""
        self._running = False
        logger.info("Event processor stopped")

class QualityAwareTransformer(nn.Module):
    """2-layer transformer encoder for assessment embeddings with quality weights"""
    
    def __init__(self, input_dim: int = 10, hidden_dim: int = 64, output_dim: int = 32):
        super().__init__()
        self.input_dim = input_dim
        self.hidden_dim = hidden_dim
        self.output_dim = output_dim
        
        # Transformer encoder layers
        encoder_layer = nn.TransformerEncoderLayer(
            d_model=input_dim,
            nhead=2,
            dim_feedforward=hidden_dim,
            dropout=0.1,
            batch_first=True
        )
        self.encoder = nn.TransformerEncoder(encoder_layer, num_layers=2)
        
        # Output projection
        self.fc = nn.Linear(input_dim, output_dim)
        self.dropout = nn.Dropout(0.1)

    def forward(self, x: torch.Tensor, quality_weights: torch.Tensor) -> torch.Tensor:
        """
        Forward pass with quality-weighted attention
        
        Args:
            x: Input tensor [batch_size, seq_len, input_dim]
            quality_weights: Quality weights [batch_size, seq_len]
        
        Returns:
            Output embeddings [batch_size, output_dim]
        """
        # Apply quality weights
        weighted_x = x * quality_weights.unsqueeze(-1)
        
        # Transformer encoding
        encoded = self.encoder(weighted_x)
        
        # Global average pooling
        pooled = encoded.mean(dim=1)
        
        # Final projection
        output = self.fc(self.dropout(pooled))
        
        return output

class TemporalConvolutionalNetwork(nn.Module):
    """3-layer TCN for conversation history processing"""
    
    def __init__(self, input_dim: int = 384, hidden_dim: int = 128, output_dim: int = 32):
        super().__init__()
        self.input_dim = input_dim
        self.hidden_dim = hidden_dim
        self.output_dim = output_dim
        
        # 3-layer convolution with residual connections
        self.conv1 = nn.Conv1d(input_dim, hidden_dim, kernel_size=3, padding=1, dilation=1)
        self.conv2 = nn.Conv1d(hidden_dim, hidden_dim, kernel_size=3, padding=2, dilation=2)
        self.conv3 = nn.Conv1d(hidden_dim, output_dim, kernel_size=3, padding=4, dilation=4)
        
        self.relu = nn.ReLU()
        self.dropout = nn.Dropout(0.1)
        self.layer_norm1 = nn.LayerNorm(hidden_dim)
        self.layer_norm2 = nn.LayerNorm(hidden_dim)
        
        # Residual connections
        self.residual1 = nn.Conv1d(input_dim, hidden_dim, kernel_size=1) if input_dim != hidden_dim else None
        self.residual2 = nn.Conv1d(hidden_dim, output_dim, kernel_size=1) if hidden_dim != output_dim else None

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        """
        Forward pass through TCN
        
        Args:
            x: Input tensor [batch_size, seq_len, input_dim]
        
        Returns:
            Output tensor [batch_size, output_dim]
        """
        # Transpose for conv1d: [batch_size, input_dim, seq_len]
        x = x.transpose(1, 2)
        
        # First layer with residual
        conv1_out = self.relu(self.conv1(x))
        if self.residual1:
            conv1_out = conv1_out + self.residual1(x)
        conv1_out = self.layer_norm1(conv1_out.transpose(1, 2)).transpose(1, 2)
        conv1_out = self.dropout(conv1_out)
        
        # Second layer with residual
        conv2_out = self.relu(self.conv2(conv1_out))
        conv2_out = conv2_out + conv1_out  # Residual
        conv2_out = self.layer_norm2(conv2_out.transpose(1, 2)).transpose(1, 2)
        conv2_out = self.dropout(conv2_out)
        
        # Third layer with optional residual
        conv3_out = self.conv3(conv2_out)
        if self.residual2:
            conv3_out = conv3_out + self.residual2(conv2_out)
        
        # Global average pooling
        output = conv3_out.mean(dim=2)
        
        return output

class DMPVNService:
    """Dynamic Multidimensional Personality Vector Network Service"""
    
    def __init__(self, config):
        self.config = config
        self.device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
        
        # Initialize models
        self.assessment_embedder = QualityAwareTransformer().to(self.device)
        self.tcn = TemporalConvolutionalNetwork().to(self.device)
        
        # Heterogeneous graph convolution
        self.hetero_conv = HeteroConv({
            ("user", "takes", "assessment"): GATConv((32, 32), 32, heads=2, concat=False)
        }).to(self.device)
        
        # Set models to eval mode (training handled separately)
        self.assessment_embedder.eval()
        self.tcn.eval()
        self.hetero_conv.eval()
        
        logger.info(f"DMPVN initialized on device: {self.device}")
        # Note: Pre-trained weights can be loaded via checkpoint path in future iterations

    async def compute_user_vector(self, user_id: str, session: AsyncSession, 
                                conversation_history: Optional[List[str]] = None) -> torch.Tensor:
        """
        Compute dynamic user vector from assessments and conversation history
        
        Args:
            user_id: User identifier
            session: Database session
            conversation_history: Recent conversation messages
        
        Returns:
            User vector tensor [32,]
        """
        try:
            # Fetch user assessments
            assessments_stmt = select(UserAssessment).where(UserAssessment.user_id == user_id)
            assessments_result = await session.execute(assessments_stmt)
            assessments = assessments_result.scalars().all()
            
            # If no conversation history provided, fetch from database
            if conversation_history is None:
                conv_stmt = select(ConversationHistory.message).where(
                    ConversationHistory.user_id == user_id
                ).order_by(ConversationHistory.timestamp.desc()).limit(30)
                conv_result = await session.execute(conv_stmt)
                conversation_history = [row[0] for row in conv_result.fetchall()]
            
            # Build heterogeneous graph
            data = HeteroData()
            
            # User node (single node per user)
            data["user"].x = torch.zeros(1, 32, device=self.device)
            
            # Assessment nodes
            if assessments:
                # Prepare assessment data
                scores_list = []
                quality_weights_list = []
                
                for assessment in assessments:
                    scores = assessment.scores_list
                    if len(scores) < 10:
                        # Pad scores to minimum length
                        scores.extend([0.0] * (10 - len(scores)))
                    scores_list.append(scores[:10])  # Take first 10 dimensions
                    
                    # Get quality weight from config
                    weight = self.config.therapeutic["test_configs"].get(
                        assessment.test_type, 
                        {"initial_weight": 0.5}
                    )["initial_weight"]
                    quality_weights_list.append(weight)
                
                # Convert to tensors
                scores_tensor = torch.tensor(scores_list, dtype=torch.float32, device=self.device)
                quality_weights = torch.tensor(quality_weights_list, dtype=torch.float32, device=self.device)
                
                # Reshape for transformer: [1, num_assessments, 10]
                scores_tensor = scores_tensor.unsqueeze(0)
                quality_weights = quality_weights.unsqueeze(0)
                
                # Generate assessment embeddings
                with torch.no_grad():
                    assessment_embeddings = self.assessment_embedder(scores_tensor, quality_weights)
                    data["assessment"].x = assessment_embeddings.squeeze(0)
                
                # Create edges: user -> assessments
                num_assessments = len(assessments)
                edge_index = torch.tensor([
                    [0] * num_assessments,  # All edges from user node 0
                    list(range(num_assessments))  # To assessment nodes
                ], dtype=torch.long, device=self.device)
                
                # Edge attributes (user confidence scores)
                edge_attr = torch.tensor([
                    a.user_confidence for a in assessments
                ], dtype=torch.float32, device=self.device)
                
            else:
                # No assessments - create dummy nodes
                data["assessment"].x = torch.zeros(1, 32, device=self.device)
                edge_index = torch.tensor([[0], [0]], dtype=torch.long, device=self.device)
                edge_attr = torch.ones(1, device=self.device)
            
            # Apply conversation context modulation
            if conversation_history:
                try:
                    # Embed conversation history
                    conv_embeddings = []
                    for message in conversation_history[-30:]:  # Last 30 messages
                        embedding = self.config.embedding_model.embed(message)
                        conv_embeddings.append(embedding)
                    
                    if conv_embeddings:
                        conv_tensor = torch.tensor(conv_embeddings, dtype=torch.float32, device=self.device)
                        conv_tensor = conv_tensor.unsqueeze(0)  # [1, seq_len, 384]
                        
                        # Generate context vector
                        with torch.no_grad():
                            context_vector = self.tcn(conv_tensor)  # [1, 32]
                        
                        # Compute modulation factor
                        modulation = torch.sigmoid(context_vector).mean()
                        
                        # Apply dynamic edge modulation
                        edge_attr = edge_attr * modulation
                        
                        logger.info(f"Applied context modulation: {modulation.item():.4f}")
                
                except Exception as e:
                    logger.warning(f"Failed to apply context modulation: {e}")
                    # Continue without modulation
            
            # Set graph edges
            data["user", "takes", "assessment"].edge_index = edge_index
            data["user", "takes", "assessment"].edge_attr = edge_attr
            
            # Compute final user vector using heterogeneous convolution
            with torch.no_grad():
                x_dict = {
                    "user": data["user"].x,
                    "assessment": data["assessment"].x
                }
                edge_index_dict = {
                    ("user", "takes", "assessment"): data["user", "takes", "assessment"].edge_index
                }
                
                # Note: HeteroConv doesn't directly support edge attributes in this version
                # Using node features only for now
                out_dict = self.hetero_conv(x_dict, edge_index_dict)
                user_vector = out_dict["user"][0]  # Get first (and only) user node
            
            logger.info(f"Computed user vector for {user_id}: shape {user_vector.shape}")
            return user_vector
            
        except Exception as e:
            logger.error(f"Failed to compute user vector for {user_id}: {e}")
            # Return default vector
            return torch.zeros(32, device=self.device)

def parse_llm_output(output: str) -> Tuple[str, str]:
    """Parse LLM output to extract thought and response"""
    thought_match = re.search(r'<thought>(.*?)</thought>', output, re.DOTALL)
    response_match = re.search(r'<response>(.*?)</response>', output, re.DOTALL)
    
    thought = thought_match.group(1).strip() if thought_match else ""
    response = response_match.group(1).strip() if response_match else output.strip()
    
    return thought, response

class SafetyGovernor:
    """Real-time safety framework for response arbitration"""
    
    def __init__(self, context):
        self.context = context
        
        # Initialize NLI model for contradiction detection
        try:
            self.nli_model = pipeline(
                "text-classification", 
                model="cross-encoder/nli-distilroberta-base",
                device=0 if torch.cuda.is_available() else -1
            )
            logger.info("NLI model loaded successfully")
        except Exception as e:
            logger.error(f"Failed to load NLI model: {e}")
            self.nli_model = None
        
        # Crisis detection keywords
        self.crisis_keywords = [
            "harm", "hurt", "kill", "suicide", "die", "death", "end it", "give up",
            "hopeless", "worthless", "better off dead", "can't go on", "want to die"
        ]
        
        # Safety thresholds from config
        self.contradiction_threshold = context.config.therapeutic["safety_thresholds"]["contradiction_threshold"]
        self.crisis_confidence_threshold = context.config.therapeutic["safety_thresholds"]["crisis_confidence_threshold"]
        
        logger.info("SafetyGovernor initialized")

    async def arbitrate_response(self, response: str, facts: List[Dict[str, Any]], user_id: str) -> str:
        """
        Arbitrate response for safety and accuracy
        
        Args:
            response: Generated therapeutic response
            facts: Retrieved factual context
            user_id: User identifier
            
        Returns:
            Safe, validated response
        """
        try:
            # Step 1: Crisis detection
            juncture, crisis_confidence = self.detect_juncture(response)
            
            # Step 2: NLI grounding for contradiction detection
            contradiction_scores = []
            if self.nli_model and facts:
                for fact in facts:
                    try:
                        nli_input = f"{response} [SEP] {fact['text']}"
                        result = self.nli_model(nli_input)
                        
                        # Extract contradiction score
                        contradiction_score = 0.0
                        for pred in result:
                            if pred["label"] == "CONTRADICTION":
                                contradiction_score = pred["score"]
                                break
                        
                        contradiction_scores.append(contradiction_score)
                        
                    except Exception as e:
                        logger.warning(f"NLI check failed for fact: {e}")
                        contradiction_scores.append(0.0)
            
            max_contradiction = max(contradiction_scores, default=0.0)
            
            # Step 3: Safety decision
            safety_override = False
            override_reason = None
            
            if max_contradiction > self.contradiction_threshold:
                safety_override = True
                override_reason = f"High contradiction score: {max_contradiction:.3f}"

            client_turns = [turn for turn in turns if turn['speaker'] == 'client']
            therapist_turns = [turn for turn in turns if turn['speaker'] == 'therapist']
            
            if not client_turns or not therapist_turns:
                logger.warning(f"Dialogue {dialogue_id} missing client or therapist turns")
                return {'dialogue_id': dialogue_id, 'status': 'skipped', 'reason': 'missing_speakers'}
            
            # Get the last client message and corresponding expert response
            last_client_turn = client_turns[-1]
            last_expert_turn = therapist_turns[-1]
            
            client_message = last_client_turn['text']
            expert_response = last_expert_turn['text']
            
            # Get AI response via API
            ai_response = await self._get_ai_response(client_message, dialogue_id)
            
            # Generate critique using LLM
            critique_result = await self._generate_critique(client_message, ai_response, expert_response)
            
            return {
                'dialogue_id': dialogue_id,
                'status': 'completed',

        Args:
            user_id: User identifier
            model_name: Base model name
            
        Returns:
            Assigned model version or None if no models available
        """
        try:
            # Get available versions for this model
            available_versions = [
                key for key in self.models.keys() 
                if key.startswith(f"{model_name}:") and self.models[key]["active"]
            ]
            
            if not available_versions:
                logger.warning(f"No active versions found for model {model_name}")
                return None
            
            # Hash-based assignment for consistent user experience
            user_hash = int(hashlib.md5(user_id.encode()).hexdigest(), 16)
            assigned_version = available_versions[user_hash % len(available_versions)]
            
            # Cache assignment
            self.user_assignments[user_id] = assigned_version
            
            logger.info(f"Assigned user {user_id} to model {assigned_version}")
            return assigned_version
            
        except Exception as e:

def create_llm_adapter(provider: str, config: Dict[str, Any]) -> LLMAdapter:
    """Factory function to create appropriate LLM adapter"""
    adapters = {
        "mock": MockAdapter,
        "openai": OpenAIAdapter,
        "huggingface": HuggingFaceLocalAdapter,
        "huggingface_local": HuggingFaceLocalAdapter
    }
    
    adapter_class = adapters.get(provider.lower())
    if not adapter_class:
        logger.warning(f"Unknown provider '{provider}', falling back to mock")
        adapter_class = MockAdapter
    
    adapter = adapter_class(config)
    
    if not adapter.validate_config():
        logger.warning(f"Invalid config for {provider}, falling back to mock")
        adapter = MockAdapter(config)
    
    logger.info(f"Created LLM adapter: {adapter.__class__.__name__}")
    return adapter
